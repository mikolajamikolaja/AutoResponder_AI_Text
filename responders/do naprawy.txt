## 1. Plik: `responders/zwykly_psychiatryczny_raport.py`

W tym pliku kluczowe jest uodpornienie systemu na błędy parsowania (gdy model pisze prozą zamiast JSON-em) oraz mechanizm ponawiania próby.

### Funkcja: `_strip_json_markdown` (Naprawia BUG 4)
*Wycina wszystko, co nie jest JSON-em (np. wiodące przecinki, tagi markdown).*

```python
def _strip_json_markdown(raw: str) -> str:
    if not raw: return ""
    # Szukamy wszystkiego między pierwszą a ostatnią klamrą/nawiasem
    match = re.search(r'(\{.*\}|\[.*\])', raw, re.DOTALL)
    if match:
        return match.group(1)
    # Jeśli brak klamer, czyścimy z markdownu i białych znaków
    clean = raw.strip().lstrip("`, \n\t")
    if clean.lower().startswith("json"):
        clean = clean[4:].strip()
    return clean
```

### Funkcja: `_parse_json_safe` (Naprawia BUG 1)
*Jeśli model „odleci” i wyśle zwykły tekst, ta funkcja nie wywali błędu, tylko zapakuje tekst w bezpieczny słownik.*

```python
def _parse_json_safe(raw: str, section: str):
    if not raw or len(raw.strip()) < 2:
        return {}
    
    clean = _strip_json_markdown(raw)
    
    try:
        return json.loads(clean)
    except json.JSONDecodeError:
        # Fallback: jeśli to czysty tekst, zapakuj go jako treść
        if len(clean) > 5 and "{" not in clean:
            logger.warning(f"[psych-raport] Sekcja {section} to surowy tekst -> wrapping")
            return {"content": clean}
        
        # Próba ratowania uciętego JSONa
        if not clean.endswith("}"):
            try: return json.loads(clean + "}")
            except: pass
        logger.warning(f"[psych-raport] Krytyczny błąd JSON w sekcji {section}")
        return {"error": "Błąd struktury", "raw": clean[:200]}
```

### Funkcja: `_call_with_retry` (Naprawia BUG 3)
*Jeśli odpowiedź jest ucięta (za krótka), funkcja automatycznie pyta model jeszcze raz z większym limitem tokenów.*

```python
def _call_with_retry(system, user, max_tokens=1000):
    res = call_deepseek(system, user, MODEL_TYLER, max_tokens=max_tokens)
    # Jeśli odpowiedź jest podejrzanie krótka (np. tylko klamra otwierająca)
    if not res or len(res.strip()) <= 5:
        logger.warning(f"[psych-raport] Odpowiedź ucięta, retry z 2x max_tokens")
        res = call_deepseek(system, user, MODEL_TYLER, max_tokens=max_tokens * 2)
    return res
```

### Dlaczego to zadziała?
1.  **Regex w `_strip_json_markdown`** ignoruje wszystko, co model wypisze „od siebie” przed właściwym obiektem JSON (w tym nieszczęsne przecinki).
2.  **`_call_with_retry`** to Twoja polisa na wypadek, gdyby DeepSeek „zasnął” w połowie generowania.
3.  **`_get_safe_photo`** eliminuje błędy `NoneType`, które wywalają cały skrypt, gdy serwer Fluxa nie odpowie na czas.
